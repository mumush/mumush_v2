Why Apache?

Apache was chosen because of its wide availability and ease of use. The other options are good, but Apache covers the widest range of use cases for most people. Also Apache has a vast user base with many users that can offer their own insight.

Preparation

APR - There are a few requirements needed to install Apache on Linux. The Apache Portable Runtime or APR and APR-Util need to be installed if not already. Just download them and extract them in ./srclib/apr and ./srclib/apr-util respectively.

Time - Make Sure that your server keeps accurate timing. Parts of the HTTP protocol rely on the time of day so if there is a problem it could mess up the site.

Disk Space - You need 50MB of space for installation and after that you only need 10MB. You may need additional space for additional modules.

PCRE- Make Sure You also have the Perl-Compatible Regular Expressions Library installed. Your linux distro should have a port or package pre made for it if not already installed. Just Google it.

Installing

Depending on your version of Linux the command to install Apache will either be:

sudo apt-get install apache2

Or this:

# yum install httpd

There is also the harder way for those that neither of those options work for. This will work on all Unix based machines.

Running the following commands will download and install Apache on your machine.

$ lynx http://httpd.apache.org/download.cgi
$ gzip -d httpd-NN.tar.gz
$ tar xvf httpd-NN.tar
$ cd httpd-NN
$ ./configure --prefix=PREFIX
$ make
$ make install

Optimization

To optimize Apache and and make it work to your needs you need to edit the configuration file.

$ vi PREFIX/conf/httpd.conf

If you need help with anything the Apache site has extensive documentation on all things Apache.

Running

There are two ways to start the server depending on which method you installed it with.

1st way:

# /etc/init.d/apache2 start

2nd way:

# /etc/init.d/httpd start

Or if you did the harder method:

$ PREFIX/bin/apachectl -k start

Finally you want to test if it's running properly by going to the site below on your server.

http://localhost/
